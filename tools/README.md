# Splash animation tools

Four-stage pipeline that converts clawd-tank SVGs into the C header the
firmware compiles in.

## Source

SVGs are vendored in `tools/svg_pipeline/svg/` from the
[clawd-tank](https://github.com/marciogranzotto/clawd-tank) set (MIT).
Active animations and their parameters are listed in
`tools/svg_pipeline/animations.json` — edit there to add, remove, or
retune an animation. The `group` field is a human-readable annotation of
intended usage-rate bucket; it is NOT propagated to the generated header and
has no runtime effect. The authoritative sync point is each animation's `name`
string matching `GROUP_NAMES` in `firmware/src/splash.cpp`.

## Pipeline

```
make_wrappers.py        SVG → tight-cropped HTML wrappers (via cairosvg)
capture_frames.mjs      Headless Playwright captures N frames per animation
frames_to_data.py       Frames → binary manifest.json + clawd_*.bin → tools/svg_anim_data/
gen_splash_header.py    Binary data → firmware/src/splash_animations.h
```

All four stages are driven by `tools/svg_pipeline/build.sh`. Stage 4 also runs
automatically as a PlatformIO pre-build hook (`firmware/scripts/gen_splash.py`) on
every `pio run` — no manual step needed to build or flash.

`firmware/src/splash_animations.h` is a **gitignored build artifact**. The committed
source of truth is the binary data in `tools/svg_anim_data/` (`manifest.json` +
`clawd_*.bin`). `gen_splash_header.py` reads those and emits:
- `splash_<ident>_f{N}[W*H]` flat arrays + `splash_<ident>_frames[N]` pointer table — per-frame palette indices
- `splash_<ident>_holds[N]` — per-frame hold time in ms
- `splash_anims[]` — master table (name, category, width, height, frame_count, palette, frames, holds)
- `SPLASH_ANIM_COUNT`, `SPLASH_PALETTE_SIZE`

The firmware (`splash.cpp`) consumes this header.

## Dependencies

- Python: `pillow`, `cairosvg` (stages 1, 3, 4)
- Node: `playwright` + a Chromium browser (`npx playwright install chromium`) — **capture stage only**; not required to build/flash firmware

## Re-running

```bash
tools/svg_pipeline/build.sh   # regenerates svg_anim_data/ + splash_animations.h
pio run -d firmware -t upload  # header auto-regenerated by pre-build hook if needed
```

The build script is idempotent — re-run whenever SVGs or animation params change.

## Icons

`tools/png_to_lvgl.js` converts Lucide PNGs to RGB565 / RGB565A8 C arrays.
See the root README's "Converting Lucide icons" section for usage.
