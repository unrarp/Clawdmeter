# splash_animations.h is a build artifact, not a committed file

## Context
`firmware/src/splash_animations.h` is a 11 MB generated C header (palette
arrays + per-frame `uint8_t` cells) produced by the SVG→capture→binary→header
pipeline. Historically the project committed both the header and the source
JSON, totalling ~26 MB per migration. The file had to be manually excluded from
code review (binary data as C arrays is unreadable) and re-generated after any
animation change.

## Decision / Solution
`splash_animations.h` is now gitignored and regenerated by a PlatformIO
`pre:` extra-script hook (`firmware/scripts/gen_splash.py`) before every
`pio run`. The hook is registered in all three `[env:...]` blocks in
`firmware/platformio.ini`:

```
extra_scripts = pre:scripts/gen_splash.py
```

`gen_splash.py` resolves the repo root via `env["PROJECT_DIR"]` (SCons context
— `__file__` is not defined in PlatformIO extra-scripts), inserts
`tools/svg_pipeline` on `sys.path`, and calls `gen_splash_header.needs_regen()`
/ `generate()`. If `tools/svg_anim_data/manifest.json` is absent it prints a
clear message and returns without calling `generate()` — avoiding the `sys.exit`
traceback a missing-data path would otherwise produce inside SCons.

Committed source of truth: `tools/svg_anim_data/manifest.json` (catalog +
palette) + `tools/svg_anim_data/clawd_*.bin` (raw uint8 palette indices,
frame-major row-major, 20×128×128 = 327,680 bytes each). Total committed: ~3.8 MB.

## Why
- **Repo size.** Generated header (~11 MB as C array text) + JSON source
  (~15 MB) = ~26 MB per commit, growing with each animation change. Binary
  + gitignored header = ~3.8 MB.
- **Binary data is not git-diffable.** Frame data as C text arrays has no
  meaningful diff. Binary `.bin` files at least keep the repo honest about
  what is diffable.
- **Analogy with `dist/`.** The header is the firmware's `dist/` — a
  TypeScript or Python project does not commit compiled output. PlatformIO's
  `extra_scripts` is the pre-build equivalent of `npm run build`.
- **No manual step on fresh checkout.** `pio run` triggers the hook; the
  stdlib-only Python generator (`gen_splash_header.py`) requires only
  PlatformIO's bundled Python — no node, no extra installs.

## Alternatives considered
- **Commit the header, gitignore the JSON.** Simpler (zero build step). Rejected:
  still grows the repo 11 MB per migration and commits machine output.
- **Commit as JSON (no binary).** Already the previous state. Rejected for
  the same size reason (~15 MB JSON is verbose; 3.8 MB binary is identical data).
- **Keep the node `convert_to_c.js` converter.** It required node to be installed
  for every build/flash. The stdlib-only Python generator removes that dependency
  from the flash path entirely.

## Prevention
- `firmware/src/splash_animations.h` is in `.gitignore`. `git status` will not
  show it after it is regenerated.
- `git rm --cached firmware/src/splash_animations.h` was run to untrack it from
  the index (gitignoring a tracked file without `rm --cached` has no effect).
- The pre-build hook prints `[splash] regenerating splash_animations.h` or
  `[splash] splash_animations.h up to date` on every build so the state is
  always visible in the build log.
